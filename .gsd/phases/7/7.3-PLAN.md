# Plan 7.3: Pagination (Growing Limit + Refresh)

## Objective
Replace the hard-coded `limit(50)` in `subscribeToEmails` with a stateful limit driven from `DashboardApp`. Add a "Load more" button that increases the limit by 20, and a refresh icon that resets back to 20. Real-time updates are preserved across the full visible set because a single `onSnapshot` covers the whole limit.

    Change `subscribeToEmails` to accept a `limitCount` parameter with a default of 20:

    ```js
    export function subscribeToEmails(userId, callback, onError, limitCount = 20) {
      const q = query(
        collection(db, "users", userId, "emails"),
        orderBy("sentAt", "desc"),
        limit(limitCount)
      );
      return onSnapshot(q,
        (snapshot) => {
          const emails = snapshot.docs.map(d => ({ id: d.id, ...d.data() }));
          callback(emails);
        },
        (error) => {
          console.error("Gmail Intel: subscribeToEmails error", error);
          onError?.(error);
        }
      );
    }
    ```

    No other changes to `db.js`.
