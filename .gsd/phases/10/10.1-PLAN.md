# Plan 10.1: Reactive Sidebar Offset via ResizeObserver on role="main"

## Objective

Replace the 250ms `setInterval` polling in `updateSidebarOffset` with a
`ResizeObserver` on Gmail's `[role="main"]` element. This element reliably
resizes whenever the Gmail side panel opens or closes, making it the correct
anchor. No MutationObserver needed — `[role="main"]` always exists after
Gmail loads.

## Measured data (from user's console)

```
role="main" open:   { x:72, width:1384 } → right edge = 1456
role="main" closed: { x:72, width:1424 } → right edge = 1496

[aria-label="Side panel"] open:   width: 56
[aria-label="Side panel"] closed: width: 16  (does NOT go display:none)
```

## Correct offset formula

```js
sidebar.style.right = (window.innerWidth - main.getBoundingClientRect().right) + 'px'
```

When side panel open:   right = viewportWidth - 1456 = ~56px offset
When side panel closed: right = viewportWidth - 1496 = ~16px offset

  <files>src/sidebar.js</files>
  <action>
    Remove the existing `updateSidebarOffset` function and its `setInterval`
    call (the block starting with `const updateSidebarOffset = () => {` through
    `setInterval(updateSidebarOffset, 250);`).

    Replace with this block:

    ```js
    // Compute sidebar right offset from Gmail's main content area.
    // role="main" resizes when the Gmail side panel opens/closes, so
    // watching it with ResizeObserver gives immediate, lag-free updates.
    const updateSidebarOffset = () => {
      const main = document.querySelector('[role="main"]');
      if (!main) return;
      const { right } = main.getBoundingClientRect();
      sidebar.style.right = Math.max(0, window.innerWidth - right) + 'px';
    };

    const main = document.querySelector('[role="main"]');
    if (main) {
      const ro = new ResizeObserver(updateSidebarOffset);
      ro.observe(main);
    }

    window.addEventListener('resize', updateSidebarOffset);
    updateSidebarOffset();
    ```

    Do NOT keep any `setInterval` calls.
    Do NOT observe `[aria-label="Side panel"]` — it is unreliable (may be
    display:none or collapsed, causing getBoundingClientRect to return zeros).
  </action>
  <verify>grep -c "setInterval" src/sidebar.js</verify>
  <done>Output is 0 (no setInterval calls remain). ResizeObserver block is present.</done>
</task>
