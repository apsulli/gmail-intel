# Plan 11.1 — Store Sender Identity on Email Docs

## Objective
Persist `fromEmail` and `fromName` on every tracked email written to Firestore, so the dashboard can group and display by sender.

## Files Changed
- `src/content.js` — pass sender fields to `logEmailSent`
- `src/api/db.js` — include `fromEmail`/`fromName` in the Firestore write

## Steps

### 1. `src/content.js` — pass sender to logEmailSent

In `handleTrackedSend`, the `user` object (Firebase auth `currentUser`) is already in scope.
Add `fromEmail` and `fromName` to the `logEmailSent` call:

```js
await logEmailSent({
  emailId: emailId,
  userId: user.uid,
  subject: subject,
  recipients: recipientLogs,
  fromEmail: user.email ?? '',
  fromName: user.displayName ?? null,
});
```

### 2. `src/api/db.js` — include in Firestore write

`logEmailSent` already spreads `...rest` into the doc. Since `fromEmail` and `fromName`
are part of the `emailData` payload (after destructuring `emailId` and `userId`), they
are automatically included in the `setDoc` write. No structural change needed — just
verify the spread covers them (it does).

## Verification
- Send a tracked email
- Check Firestore console: `users/{uid}/emails/{emailId}` should have `fromEmail` and `fromName` fields
- Existing docs without these fields are unaffected (dashboard falls back to "Unknown Sender")
