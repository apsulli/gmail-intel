# Plan 9.1: Seen State Module + Bold Unread Email Rows

## Objective

Introduce a `seen` state layer: emails with opens/clicks are displayed with a
**bold subject** in the dashboard until the user expands (reads) them. "Seen"
state persists across page reloads via `chrome.storage.local`.

## Context

- `src/dashboard/DashboardApp.jsx` — EmailRow and DashboardApp components
- `src/api/seen.js` — NEW file to create

## Tasks

<task type="auto">
  <name>Create src/api/seen.js — seen state helpers</name>
  <files>src/api/seen.js</files>
  <action>
    Create a NEW file `src/api/seen.js` with two exported async functions:

    ```js
    // src/api/seen.js

    export async function markSeen(emailId) {
      const data = await chrome.storage.local.get('seenEmails');
      const seen = data.seenEmails ?? {};
      seen[emailId] = true;
      await chrome.storage.local.set({ seenEmails: seen });
    }

    export async function getSeenMap() {
      const data = await chrome.storage.local.get('seenEmails');
      return data.seenEmails ?? {};
    }
    ```

    No other helpers needed. Keep it minimal.
  </action>
  <verify>grep -l "markSeen" src/api/seen.js</verify>
  <done>File exists and exports markSeen and getSeenMap.</done>
</task>
