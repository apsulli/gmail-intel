# Plan 9.2: Extension Icon Badge Count

## Objective

Show a live badge count (pink background) on the extension icon whenever emails
have unread opens or clicks. The count is the number of emails with activity the
user hasn't expanded yet. Clears when all emails are expanded/seen.

Depends on Plan 9.1 (seenMap + isSeen in place).

## Context

- `src/background.js` — badge API lives here (chrome.action only works in SW)
- `src/dashboard/DashboardApp.jsx` — computes unread count, sends message
- `src/api/seen.js` — already created in 9.1

<task type="auto">
  <name>Compute unread count in DashboardApp and update badge</name>
  <files>src/dashboard/DashboardApp.jsx</files>
  <action>
    **Approach:** Each EmailRow already knows its own open/click counts (via its
    events subscription). We use an `onUnreadChange(emailId, isUnread)` callback
    to propagate this to DashboardApp, which aggregates and sends the badge count.

    **1. In DashboardApp, add unreadSet state:**
    ```js
    const [unreadSet, setUnreadSet] = useState(new Set());
    ```

    **2. Add onUnreadChange handler in DashboardApp:**
    ```js
    const handleUnreadChange = (emailId, isUnread) => {
      setUnreadSet(prev => {
        const next = new Set(prev);
        if (isUnread) next.add(emailId);
        else next.delete(emailId);
        return next;
      });
    };
    ```

    **3. Add useEffect to send badge count whenever unreadSet changes:**
    ```js
    useEffect(() => {
      chrome.runtime.sendMessage({ type: 'SET_BADGE', count: unreadSet.size });
    }, [unreadSet]);
    ```

    **4. Pass onUnreadChange to EmailRow:**
    In the EmailRow render call, add:
    ```jsx
    onUnreadChange={handleUnreadChange}
    ```

    **5. In EmailRow component**, add `onUnreadChange` to destructured props:
    ```js
    function EmailRow({ email, userId, onSelect, selected, isSeen, onUnreadChange }) {
    ```

    **6. In EmailRow, report unread status when events change:**
    Add this useEffect AFTER the existing events useEffect (the one that calls
    subscribeToEvents):
    ```js
    useEffect(() => {
      const hasActivity = opens > 0 || clicks > 0;
      onUnreadChange?.(email.id, hasActivity && !isSeen);
    }, [opens, clicks, isSeen]);
    ```

    Note: `opens` and `clicks` are already computed from events (lines ~81-82 in
    the original file). This effect depends on them directly.

    **7. Clear badge on unmount (dashboard closed):**
    In DashboardApp, add a cleanup to the unreadSet effect:
    ```js
    useEffect(() => {
      chrome.runtime.sendMessage({ type: 'SET_BADGE', count: unreadSet.size });
      return () => {
        // Clear badge when dashboard unmounts
        chrome.runtime.sendMessage({ type: 'SET_BADGE', count: 0 });
      };
    }, [unreadSet]);
    ```
    Replace the simpler version from step 3 with this version that includes cleanup.

    Do NOT add the cleanup as a separate useEffect — it must be the return value
    of the same effect that sends the badge update.
  </action>
