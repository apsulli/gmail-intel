# Summary 9.1: Seen State + Bold Unread Email Rows

## What Was Done

### New file: `src/api/seen.js`
Created a minimal chrome.storage.local wrapper with two exports:
- `markSeen(emailId)` ‚Äî writes `{ seenEmails: { [emailId]: true } }` to storage
- `getSeenMap()` ‚Äî reads and returns the full seen map (defaults to `{}`)

### Changes to `src/dashboard/DashboardApp.jsx`

**EmailRow component:**
- Added `isSeen` and `onUnreadChange` to destructured props
- Added `useEffect` that fires when `opens`/`clicks`/`isSeen` change ‚Äî calls `onUnreadChange(email.id, hasActivity && !isSeen)`
- Subject `<span>` fontWeight is now `700` when email has activity and `!isSeen`, otherwise `500`

**DashboardApp component:**
- Imported `markSeen` and `getSeenMap` from `../api/seen.js`
- Added `seenMap` state, loaded from storage on mount via `useEffect`
- Added `unreadSet` state (a `Set` of emailIds with unseen activity)
- Added `handleSelect(email)` ‚Äî calls `markSeen` + updates local `seenMap` state on expand
- Added `handleUnreadChange(emailId, isUnread)` ‚Äî maintains `unreadSet`
- Added `useEffect` on `unreadSet` ‚Äî sends `SET_BADGE` to background, clears badge on unmount
- EmailRow now receives `onSelect={handleSelect}`, `isSeen={!!seenMap[email.id]}`, `onUnreadChange={handleUnreadChange}`
