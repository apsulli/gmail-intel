# Summary 9.2: Extension Icon Badge Count

## What Was Done

### Changes to `src/background.js`

Added a `SET_BADGE` message handler inside `chrome.runtime.onMessage.addListener`:
- Reads `message.count` (defaults to 0)
- Calls `chrome.action.setBadgeText({ text: count > 0 ? String(count) : '' })`
- Calls `chrome.action.setBadgeBackgroundColor({ color: '#FF1493' })` when count > 0
- Badge text cleared (empty string) when count reaches 0

### DashboardApp.jsx (already complete from 9.1)

The `unreadSet` state and `SET_BADGE` useEffect were wired in plan 9.1 as part of the same component refactor:
- `useEffect` on `unreadSet` sends `{ type: 'SET_BADGE', count: unreadSet.size }` to background
- Cleanup function sends `{ type: 'SET_BADGE', count: 0 }` when DashboardApp unmounts (sidebar closed)
- `onUnreadChange` propagates per-email unread status from EmailRow up to DashboardApp

## Data Flow

```
EmailRow events subscription
  → opens/clicks computed
  → onUnreadChange(emailId, hasActivity && !isSeen)
    → DashboardApp.unreadSet updated
      → useEffect sends SET_BADGE to background
        → chrome.action.setBadgeText (pink #FF1493)
```
