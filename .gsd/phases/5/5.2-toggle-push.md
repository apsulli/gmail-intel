# Plan 5.2 — Fix Toggle Button Push (Sidebar Open/Close)

## Problem

When the sidebar is open, the 📊 toggle button stays at `right: 16px` and sits on top
of the sidebar content, obscuring email rows near the right edge.

## Fix

Animate the toggle button's `right` position in sync with the sidebar slide animation:

- Sidebar **closed**: toggle at `right: 16px`
- Sidebar **open**: toggle at `right: 376px` (sidebar width 360px + 16px gap)

Both transitions use the same `0.3s ease` timing as the sidebar transform, so the button
appears to be "pushed" left by the opening panel.

## Changes — `src/sidebar.js`

Update the toggle click handler to also transition the toggle's right position:

```js
toggle.addEventListener('click', () => {
  const isOpen = sidebar.style.transform === 'translateX(0px)'
    || sidebar.style.transform === 'translateX(0)';

  if (isOpen) {
    sidebar.style.transform = 'translateX(100%)';
    toggle.style.right = '16px';
  } else {
    sidebar.style.transform = 'translateX(0)';
    toggle.style.right = '376px';
  }
});
```

Also add `transition` to the toggle element so its movement is smooth:

```js
Object.assign(toggle.style, {
  // ... existing styles ...
  transition: 'right 0.3s ease',
});
```

## Files Changed

- `src/sidebar.js` — add `transition: 'right 0.3s ease'` to toggle styles; update click
  handler to set `toggle.style.right` on open/close

## Verification

1. Load Gmail with extension
2. Click 📊 — sidebar slides in, toggle button smoothly moves left to sit flush with the
   sidebar's left edge
3. Click 📊 again — sidebar closes, toggle returns to bottom-right corner
4. With sidebar open, scroll email list — toggle does not overlap any content
