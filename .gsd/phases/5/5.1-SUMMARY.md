# Plan 5.1 Summary — Fix Draft Deletion via drafts.send

## What Was Done

Replaced the unreliable `DELETE_DRAFT_BY_SUBJECT` approach (subject-search after send)
with atomic `drafts.send` which sends and deletes the draft in a single API call.

### Changes Made

**`src/background.js`**
- Removed `DELETE_DRAFT_BY_SUBJECT` handler
- Added `GET_LATEST_DRAFT_ID` handler: `GET /drafts?maxResults=1`, returns most recent draft ID
- Added `SEND_DRAFT` handler: `POST /drafts/send` with `{id, message: {raw}}` body

**`src/api/gmail.js`**
- Added `draftId` parameter to `sendTrackedEmail`
- Selects `SEND_DRAFT` when `draftId` present, `SEND_EMAIL` otherwise

**`src/content.js`**
- Added `getDraftId(composeWindow)` — DOM extraction (input[name="draft"], [data-draft-id])
- Added `getLatestDraftId(token)` — API fallback via `GET_LATEST_DRAFT_ID`
- In `handleTrackedSend`: resolves `draftId` before send loop; first recipient uses it, subsequent ones get `null`
- Removed `DELETE_DRAFT_BY_SUBJECT` call
