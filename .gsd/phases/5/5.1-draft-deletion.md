# Plan 5.1 — Fix Draft Deletion via drafts.send

## Problem

The current `DELETE_DRAFT_BY_SUBJECT` approach in `background.js` searches Gmail's search
index for a draft by subject string after the send. This is unreliable because:

1. Gmail's search index has latency — a draft composed seconds ago may not be indexed yet
2. Deleting a draft via `DELETE /drafts/{id}` after using `messages.send` is a two-step
   race condition (Gmail can re-autosave between send and delete)

## Root Cause

We use `messages/send` to send our tracked email. Gmail has no way to know this corresponds
to the open compose draft. The draft persists independently.

## Fix: Switch to `drafts.send` flow

`POST /gmail/v1/users/me/drafts/send` sends the draft AND deletes it atomically. We inject
our tracked content into the request body, overriding the draft's raw content at send time.

### Step 1 — Extract draft ID from the compose window DOM

Gmail stores the draft ID in the compose window's DOM. Before doing anything else in
`handleTrackedSend`, scan these locations in order:

```js
function getDraftId(composeWindow) {
  // 1. Input with name="draft" (legacy compose popups)
  const input = composeWindow.querySelector('input[name="draft"]');
  if (input?.value) return input.value;

  // 2. data-draft-id attribute anywhere in compose
  const el = composeWindow.querySelector('[data-draft-id]');
  if (el) return el.getAttribute('data-draft-id');

  // 3. Fall back: list most recent draft via API (see Step 2)
  return null;
}
```

### Step 2 — API fallback for draft ID

If DOM extraction returns null, call a new `GET_LATEST_DRAFT_ID` background handler
that calls `GET /gmail/v1/users/me/drafts?maxResults=1` and returns the most recent
draft ID. This is safe because we call it immediately when the send button is clicked
(before any network requests), so it will be the compose draft.

Add to `background.js`:

```js
if (message.type === 'GET_LATEST_DRAFT_ID') {
  const { token } = message;
  fetch('https://gmail.googleapis.com/gmail/v1/users/me/drafts?maxResults=1', {
    headers: { Authorization: `Bearer ${token}` },
  })
  .then(r => r.json())
  .then(data => {
    const id = data.drafts?.[0]?.id ?? null;
    sendResponse({ id });
  })
  .catch(e => sendResponse({ error: e.message }));
  return true;
}
```

### Step 3 — Replace SEND_EMAIL with SEND_DRAFT

Replace the `SEND_EMAIL` background handler (which calls `messages/send`) with a new
`SEND_DRAFT` handler that calls `drafts/send`:

```js
if (message.type === 'SEND_DRAFT') {
  const { token, draftId, payload } = message;
  fetch('https://gmail.googleapis.com/gmail/v1/users/me/drafts/send', {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${token}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      id: draftId,
      message: { raw: payload },
    }),
  })
  .then(async (response) => {
    if (!response.ok) {
      const errText = await response.text();
      sendResponse({ error: `Gmail Error [${response.status}]: ${errText}` });
    } else {
      sendResponse({ data: await response.json() });
    }
  })
  .catch(e => sendResponse({ error: 'Network Error: ' + e.message }));
  return true;
}
```

Keep `SEND_EMAIL` as a fallback for when no draft ID is found (e.g., new compose opened
via a keyboard shortcut before Gmail creates the draft).

### Step 4 — Update `sendTrackedEmail` in `gmail.js`

Add a `draftId` parameter:

```js
export async function sendTrackedEmail(token, { to, subject, body, trackingPixelHtml, draftId }) {
  // ... build b64Safe as before ...

  const messageType = draftId ? 'SEND_DRAFT' : 'SEND_EMAIL';
  const messagePayload = draftId
    ? { type: messageType, token, draftId, payload: b64Safe }
    : { type: messageType, token, payload: b64Safe };

  return new Promise((resolve, reject) => {
    chrome.runtime.sendMessage(messagePayload, (response) => {
      if (chrome.runtime.lastError) return reject(new Error(chrome.runtime.lastError.message));
      if (response.error) return reject(new Error(response.error));
      resolve(response.data);
    });
  });
}
```

### Step 5 — Update `handleTrackedSend` in `content.js`

```js
// Get draft ID before sending (used to atomically send+delete draft)
const draftId = getDraftId(composeWindow) ?? await getLatestDraftId(token);

// Pass draftId through to each sendTrackedEmail call
await sendTrackedEmail(token, { to: recipient, subject, body: bodyHtml, trackingPixelHtml, draftId });
```

Note: only the FIRST recipient's send uses the draft ID (the draft is deleted after that
send). Subsequent recipients (mail merge) use `SEND_EMAIL` (draftId = null).

### Step 6 — Remove DELETE_DRAFT_BY_SUBJECT

Remove the now-obsolete `DELETE_DRAFT_BY_SUBJECT` handler from `background.js` and the
call to it in `content.js`.

### Step 7 — UI cleanup only (no draft logic)

The discard button click is kept purely for closing the compose window UI, with no
draft-deletion responsibility:

```js
// Close compose window UI only
const discardSelector = '[data-tooltip="Discard draft"], [aria-label="Discard draft"], [title="Discard draft"]';
const discardBtn = composeWindow.querySelector(discardSelector) || document.querySelector(discardSelector);
if (discardBtn) discardBtn.click();
else composeWindow.remove();
```
