# Plan 4.2: Sidebar Shell & React Mount

## Objective

Inject a persistent sidebar panel into Gmail's UI with a toggle icon button. Mount a React root into the sidebar so dashboard components (built in Plan 4.3) can render inside it.

## Context

- src/content.js — Existing content script with MutationObserver and DOM injection patterns
- src/main.jsx — Existing React entry point (used by popup, NOT the sidebar)
- src/App.jsx — Existing popup app (NOT the sidebar)
- manifest.json — Chrome extension manifest
- vite.config.js — Vite + CRXJS build config

## Tasks

<task type="auto">
  <name>Create sidebar injection module</name>
  <files>src/sidebar.js, src/content.js</files>
  <action>
    Create a new file `src/sidebar.js` that handles sidebar DOM injection. Keep it separate from `content.js` to maintain separation of concerns (content.js = tracking injection, sidebar.js = dashboard UI).

    `src/sidebar.js` should:

    1. **Create the sidebar container:**
       - Create a `div#gmail-intel-sidebar` element
       - Style it as a fixed panel on the right side of the viewport:
         - `position: fixed`, `top: 0`, `right: 0`, `height: 100vh`, `width: 360px`
         - `background: #fff`, `box-shadow: -2px 0 8px rgba(0,0,0,0.15)`
         - `z-index: 9999` (above Gmail's UI)
         - `transform: translateX(100%)` (hidden by default)
         - `transition: transform 0.3s ease`
         - `overflow-y: auto`
         - `font-family: 'Google Sans', Roboto, Arial, sans-serif`
       - When open, set `transform: translateX(0)`
       - Append to `document.body`

    2. **Create the toggle button:**
       - Create a `div#gmail-intel-toggle` element (a small icon button)
       - Style it fixed on the right edge: `position: fixed`, `right: 16px`, `top: 80px`
         - `width: 40px`, `height: 40px`, `border-radius: 50%`
         - `background: #1a73e8`, `color: white`, `cursor: pointer`
         - `box-shadow: 0 2px 8px rgba(0,0,0,0.2)`, `z-index: 10000`
         - `display: flex`, `align-items: center`, `justify-content: center`
         - `font-size: 18px`
       - Icon: Use the text "📊" (chart emoji) as a simple icon. No external icon library needed.
       - On click, toggle sidebar open/closed by adding/removing an `.open` class

    3. **Export an `initSidebar()` function** that:
       - Creates the sidebar and toggle elements
       - Appends them to the body
       - Returns the sidebar container div (so React can mount into it)

    4. **Import and call `initSidebar()` from `src/content.js`:**
       - Add `import { initSidebar } from './sidebar.js';` at the top
       - Call `initSidebar()` immediately after the "Gmail Intel Content Script Loaded" log
       - Do NOT put it inside the MutationObserver callback (sidebar is created once)

    IMPORTANT CONSTRAINTS:
    - Do NOT use Shadow DOM — keep it simple, we can add isolation later if needed
    - Do NOT import React in sidebar.js — just create the DOM container. React mounting happens in the next step.
    - Do NOT modify the existing MutationObserver or tracking toggle logic

  </action>
  <verify>npm run build — should compile with no errors</verify>
  <done>
    - `src/sidebar.js` exists with `initSidebar()` export
    - Sidebar div and toggle button are injected into Gmail on page load
    - Toggle button shows/hides the sidebar
    - Build passes
  </done>
</task>

<task type="auto">
  <name>Mount React app into sidebar</name>
  <files>src/dashboard/DashboardApp.jsx, src/dashboard/index.jsx, src/content.js</files>
  <action>
    Create a separate React entry point for the sidebar dashboard (distinct from the popup's App.jsx).

    1. **Create `src/dashboard/DashboardApp.jsx`:**
       - A placeholder React component that renders:
         ```jsx
         <div style={{ padding: '16px' }}>
           <h2 style={{ margin: '0 0 4px 0', fontSize: '16px', color: '#202124' }}>Gmail Intel</h2>
           <p style={{ margin: 0, fontSize: '12px', color: '#5f6368' }}>Tracked emails will appear here.</p>
         </div>
         ```
       - Export as default

    2. **Create `src/dashboard/index.jsx`:**
       - Export a function `mountDashboard(container)` that:
         - Creates a React root on the container element
         - Renders `<DashboardApp />` into it
       - This keeps the React mount logic self-contained

    3. **Update `src/content.js`:**
       - Import `{ mountDashboard }` from `./dashboard/index.jsx`
       - After `initSidebar()` returns the container, call `mountDashboard(container)`
       - The sidebar now has a React app rendering inside it

    IMPORTANT CONSTRAINTS:
    - Keep `DashboardApp.jsx` as a minimal placeholder — full UI is Plan 4.3
    - Do NOT modify the popup's `App.jsx` or `main.jsx`
    - The dashboard React tree is completely separate from the popup React tree

  </action>
  <verify>npm run build — should compile with no errors</verify>
  <done>
    - `src/dashboard/DashboardApp.jsx` exists
    - `src/dashboard/index.jsx` exists with `mountDashboard` export
    - React renders inside the sidebar container
    - Build passes
  </done>
</task>
