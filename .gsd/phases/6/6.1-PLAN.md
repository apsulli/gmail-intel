# Plan 6.1: Data Layer — New Firestore Paths + emailLookup

## Objective

Rewrite `src/api/db.js` to use the new multi-tenant data model:
`users/{userId}/emails/{emailId}` instead of `emails/{emailId}`.
Also write a `emailLookup/{emailId} → { userId }` document alongside
each email so Cloud Functions can resolve the user path without the
userId being embedded in tracking URLs.

## Context
- `.gsd/DECISIONS.md` — Phase 6 decisions (lookup collection, path structure)
- `src/api/db.js` — File being replaced
- `src/content.js` — Calls `logEmailSent({ emailId, userId, subject, recipients })`
- `src/dashboard/DashboardApp.jsx` — Calls `subscribeToEmails(userId, cb, onError)` and `subscribeToEvents(emailId, cb, onError)`
