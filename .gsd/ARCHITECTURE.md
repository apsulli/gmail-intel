## Overview

Gmail Intel is a **Chrome Extension** (Manifest V3) that adds email tracking capabilities to Gmail. When a user composes an email in Gmail, the extension injects a "Track Email" toggle. When tracking is enabled, the extension intercepts the native Send action, rewrites links and appends a tracking pixel, sends the email via the Gmail API, and logs metadata to Firestore. Backend Cloud Functions record open and click events as recipients interact with the tracked emails.

## Data Flow

### Sending a Tracked Email

1. User composes an email in Gmail → `MutationObserver` detects compose window
2. `injectToggle()` adds "Track Email" checkbox (checked by default)
3. User clicks Send → event intercepted, native send prevented
4. `handleTrackedSend()` fires:
   - Gets OAuth token via background service worker
   - Authenticates with Firebase using `signInWithCredential()`
   - Parses recipients from compose window DOM (inputs → chips → fallback)
   - For each recipient:
     - Generates unique `recipientId`
     - Rewrites all `<a>` hrefs to route through `trackClick` Cloud Function
     - Appends 1x1 tracking pixel `<img>` pointing to `trackPixel` Cloud Function
     - Sends individual email via Gmail API (proxied through background SW)
   - Logs metadata to Firestore `emails` collection
   - Discards the compose draft

### Tracking an Open

1. Recipient loads email → browser requests tracking pixel image
2. `trackPixel` Cloud Function logs event (`open`, recipientId, timestamp, userAgent, IP)
3. Returns transparent 1x1 GIF with no-cache headers

### Tracking a Click

1. Recipient clicks a link → hits `trackClick` Cloud Function
2. Function logs event (`click`, recipientId, targetUrl, timestamp, userAgent, IP)
3. Redirects user to the original target URL

## Technical Debt

- [ ] **Popup dashboard is a stub** — `App.jsx` renders placeholder text with no real functionality. Phase 4 of the bootstrap plan (in-Gmail React Dashboard) is incomplete.
- [ ] **No in-Gmail dashboard** — Original plan calls for injecting a React dashboard panel into the Gmail sidebar; this has not been implemented.
- [ ] **No error handling in `gmail.js`** — The `sendTrackedEmail` function uses raw `btoa(unescape(encodeURIComponent(...)))` for base64 encoding, which may fail on certain character sets.
- [ ] **Firebase config contains hardcoded API key** — `firebase-config.js` has the API key committed in plain text. While Firebase API keys are not secret in the same way as server-side keys, this is a best-practices concern.
- [ ] **No automated tests** — Zero test files found in the project.
- [ ] **Recipient parsing is fragile** — The content script relies on Gmail's internal DOM structure (`data-tooltip`, `aria-label`, `data-hovercard-id`) which can break on Gmail UI updates.
- [ ] **No rate limiting or abuse protection** — Cloud Functions `trackPixel` and `trackClick` are public HTTP endpoints with no rate limiting.
- [ ] **No logging/monitoring** — No structured logging beyond `console.log/error`.
- [ ] **CRXJS beta dependency** — `@crxjs/vite-plugin` is pinned to a beta release (`^2.0.0-beta.23`).
- [ ] **No input validation on Cloud Functions** — `trackClick` redirects to any user-supplied `targetUrl`, which is an open redirect vulnerability.

## Conventions

**Bundling:** Vite + CRXJS plugin for Chrome extension output.
